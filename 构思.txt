【项目一句话】
把「梅沃尔式」技能养成 + 资源互喂，做成能自己操控角色的 2D 游戏。
（不是挂机点一下，而是你人在场里干活、打架。）

---

【方向已定】
俯视 · 探索走动 + 经营 · 挂机=离线补进度
大世界 + 传送 · 先切屏（chunk 数据预留后期无缝）
职业/技能：梅沃尔式随时切换、任意搭配；生活技能升级解锁内容
背景：普通人 · 有魔法、科技不发达 · 世界感类梅沃尔/异世界（不要四神那套）
长期目标：一直变强（无强制通关结局）
叙事：主线当氛围，不靠剧情推游戏
战斗 ↔ 采集：约 55 开，两条都能选、都能变强
死亡：回最近存档点（村落/城镇）
仓库：与背包分开；解锁后各村镇互通，可扩容
内容类型 ✅ 已定：只要冒险，不要沙盒
  · 不是 Necesse 那种「自由建、随便改世界、沙盒定居」
  · 世界以探索、技能、战斗、既有村镇与产线推进为主
  · 地图/建筑以设计好的内容和解锁为主，不是玩家沙盘
游戏名：待定（北欧/奇幻自造地名方向，候选另附文末）
城镇建造：仅预留，以后版本——若做也须服从「冒险」而非沙盒建城
开发节奏：先做好开局一段地图+玩法，再慢慢版本更新
美术：像素风
平台：先网页单机；云存档后置（先本地+导出）

---

【核心循环 · 细化（暂定）】
短循环（几分钟）：
  出门 → 采一点资源 → 回安全点加工/卖/吃 → 再出门

中循环（一小时体感）：
  专精 1～2 个技能升级 → 解锁新配方或新工具 → 能打过 / 进得去新区域

长循环（版本内）：
  开图 → 技能全面铺开 → 装备与产线成型 → 解锁传送与更远大陆内容
  （更远的：建造城镇 = 以后版本）

长期目标 ✅ 已定：一直变强（无「通关即结束」的主目标）
  · 技能、装备、开图、产线、以后建城等，都是变强/变完整的手段
  · 不靠大结局定义玩完；内容随版本往外加即可
  · 可有阶段性里程碑（某技能满、某区域打通），但不是强制主线通关

叙事 ✅ 已定：主线故事当氛围
  · 不靠剧情推关；环境、NPC 闲聊、物品描述等点到为止
  · 不做强强制观影式主线；玩家可忽略故事仍完整游玩

玩家决策重点：
  · 现在去哪张图、练哪个技能
  · 资源留着做还是卖掉
  · 装备怎么搭（近战/远程/魔法随时可换）
  不是：复杂连招、极限走位、追主线任务

【操作与交互（暂定）】
- 移动：方向键 / WASD，俯视八向或四向均可（实现时再定）
- 互动：靠近可互动物 → 按互动键（砍、钓、挖、对话、开门、开箱）
- 生活技能操作：简单即可
  · 例：对着树按互动 → 短进度/几次敲击 → 掉木头 + 技能经验
  · 不做节奏音游式小游戏（除非以后当可选）
- 背包 ✅ 已定：格子制（不是负重）
  · 初始格子有限
  · 在商店花钱解锁扩容
  · 理论上钱够可以一直扩（无限大）；可用价格递增控制节奏
  · （实现上仍是「一次扩 N 格」，不是真无限 UI，只是不设硬顶）
- 仓库 ✅ 已定：与背包分开
  · 各村落/城镇的仓库互通（同一套库存）
  · 解锁仓库功能后，任意已开放的村/镇都能存取
  · 仓库也可扩容（花钱或其它条件，细则以后定）
  · 用途：屯材料、减轻背包压力；出门用背包，囤货进仓库
- 快捷：可切换手持工具/武器（斧、镐、钓竿、剑、弓、法杖……）
- UI 大致：血量/状态、当前技能等级摘要、背包、地图（后期）、技能面板、昼夜指示

【时间 · 昼夜 ✅ 已定】
- 有昼夜循环（影响氛围；具体是否改怪/店/钓鱼等以后再加规则）
- 没有体力条：移动、采集、战斗不靠「体力耗尽」限制

【技能体系 · 细则（暂定）】

总原则：
  · 所有技能一条角色可全练，无转职锁
  · 练什么升什么；升级主要奖励 = 解锁内容 + 小幅效率
  · 技能之间互喂，避免完全孤立

生活技能（首发建议从少到多扩，不是一次做完）：
  1 砍树（Woodcutting）
  2 挖矿（Mining）
  3 钓鱼（Fishing）
  4 烹饪（Cooking）
  5 锻造（Smithing）——矿 → 条/零件 → 工具武器
  以后可加：炼金、工艺、农业、建造（建造与城镇版挂钩，可后置）……

战斗技能（可先粗分）：
  · 近战 / 远程 / 魔法（或再拆攻防血，首发不必拆太细）
  · 经验来源：击杀、使用对应方式战斗
  · 切换：换装备即可换流派，无冷却转职

升级解锁举例（设计模板，数值以后定）：
  · 砍树 1：普通树 → 木头
  · 砍树 若干级：硬木/新树种 → 更好木板、新区域林区
  · 挖矿 若干级：新矿脉、进矿洞层数
  · 烹饪 若干级：新食谱、更高回血/增益
  · 锻造 若干级：新金属装备、更好斧镐（反哺采集效率）

经验与等级：
  · 有等级上限（可分段扩展，如先 50/99 概念，不必抄数字）
  · 效率随等级与工具小幅提升，主要爽感仍在「解锁」

【资源与经营（暂定）】
基础链示例：
  树 → 木头 →（以后）木工/箭矢/建筑材料
  矿 → 矿石 → 锻造 → 工具/武器/防具
  鱼/肉 + 其它 → 烹饪 → 食物（探索续航）
  战斗掉落 → 材料/金币 → 商店或再加工

经营感来自：
  · 缺什么就去哪条产线补
  · 工具迭代（更好斧 = 更快/能砍更高树）
  · 区域特产（某图只有某种矿/鱼）
  · 商店与配方（城镇买种子/图纸/基础补给——首发可简单）

不做（含首发与整体气质）：
  · 沙盒：自由铺地、随意改建世界、纯建造玩法
  · 复杂股市、全自动大型工厂
  · 玩家建造整座城：以后若做也只能服务冒险（据点功能），不当沙盒主轴

【战斗与采集 · 权重 ✅ 已定：约 55 开，两条都能走】
- 战斗 和 生活采集（砍树/挖矿/钓鱼等）同等重要，不是谁当调剂
- 玩家可偏战斗、偏采集，或两边都玩；两条路都能推进「一直变强」
- 理想状态：只打架也能有装备/材料/进度；只采+加工也能有钱/工具/解锁（具体数值以后调到都成立）
- 两者互喂仍在：战斗掉料可加工，采集产线可出更好武器防具与食物

【战斗形态（暂定）】
- 场地：大世界遇敌 和/或 野怪区；不强制回合制
- 形态：轻动作即时——靠近砍、远程射、放简单法术；以「能打过+换装」为主，不搞深连招
- 怪物：分区不同；强区有技能/装备门槛（提示即可）
- 掉落：材料、金币、偶发装备（保证战斗路线有收益）
- 死亡 ✅ 已定：回到最近存档点（村落或城镇）
  · 温和处理，符合经营探索；具体是否掉一点钱/耐久以后再定，原则是回安全点续玩

---

【挂机 / 离线 ✅ 已定：离线补进度】
形态：离开前在做的事，离线按时间补一部分进度（有上限）
  · 例：正在砍树/钓鱼/制作/训练某技能 → 回来结算这段时间的经验与产出（打折或封顶）
  · 不是以「据点全自动产业」为主（那种以后若加也是额外，不替代这条）

不能离线替代：
  · 开图、首通、关键战斗推进、重要解锁仍要亲自玩

原则：
  · 主体仍是上线跑图与决策
  · 离线 = 补进度、减焦虑，不能靠挂机毕业
  · 具体倍率、时长上限以后定

【世界与进度（玩法向）】
- 起点 ✅ 已定：一个小村子（安全枢纽：商店、互通仓库、简易加工、休息/存档等）
- 开局外围 ✅ 已定（切屏，示意顺序）：
    小村子 → 草地 → 河边 →（河对岸）森林
  · 河是天然分界；过河进森林（桥/浅滩/以后再定怎么过）
  · 矿区等更远内容以后再扩
- 门槛：装备、食物、技能等级、关键道具（钥匙、绳索等）
- 传送：找到并激活驿站/石阵后解锁；或技能/任务解锁回城
- 地图：先切屏；数据按 chunk；以后可无缝
- 信息：可做简单地图标记（资源点、未激活传送点）

【首发内容切片 ✅ 策略：先做好开局，再慢慢更新】
原则：只把「刚进游戏那一段」做完整，地图和系统后面版本加。

第一版建议范围（开局）：
  地图 ✅：小村子 → 草地 → 河边 → 河对岸森林（约 4 块，切屏）
    · 村子：商店（扩背包、买卖）、安全点
    · 草地：初级采集/走动过渡
    · 河边：钓鱼等；过河节点（桥/浅滩待定）
    · 对岸森林：砍树、稍深一点的探索（可作开局后半段）
  玩法：移动、互动、格子背包、商店扩容、昼夜
  技能：1～2 个生活技能（待定；河边→钓鱼、森林→砍树很顺）
  循环：出门采 → 回村卖/存/扩包 → 再出门；技能能升级并有一点解锁
  战斗：可极简一只弱怪，或再下一小步再加
  挂机：可极简，不挡开局完成度

第一版明确不做：
  全大陆、传送网、建城、无缝、全技能、大剧情、复杂魔法

之后更新顺序（示意）：
  新地图块 → 新资源/怪 → 新技能配方 → 传送 → 更完整挂机 → …… → 很久以后建城

【平台与存档】
平台 ✅ 已定：先网页单机（浏览器打开就能玩，不强制联机）
- 不先做必须在线才能玩的 MMO
- 以后要上架/包装再另说

本地存档（第一版就该有）：
  · 浏览器 localStorage 或 IndexedDB 自动存
  · 建议再加：导出/导入存档文件（换电脑、防清缓存）
  · 挂机结算用「上次在线时间戳」在本地就算得出来

云存档（想法，可后置，不挡网页单机）：
  目的：换设备继续玩、防清浏览器数据、可选多端
  原则：云是备份/同步，不是「没网就不能玩」；单机逻辑仍以本地为准

  做法分层（由易到难）：
  1. 仅导出码/文件 — 用户自己网盘备份（零后端，先做这个也行）
  2. 访客云码 — 生成一串存档码上传匿名槽（轻后端，无正式账号）
  3. 账号云存档 — 登录后自动同步（邮箱/Steam/第三方登录）
     · 冲突策略：上次写入优先，或手动选「用云端 / 用本地」
     · 频率：关键节点（回村、关闭页、每隔 N 分钟）推一次，勿每帧上传
     · 内容：存「进度数据」JSON，别存整张地图资源
  4. 以后若有挂机收菜：以服务器时间为准防改时间作弊（若你在意的话）

  建议节奏：
  · 现在：网页单机 + 本地自动存 + 导出/导入
  · 有人玩了再加：云备份（2 或 3）
  · 不必第一天就上账号系统

  注意：
  · 纯前端可被改档；单机娱乐可接受，若以后排行榜再考虑校验
  · 隐私：云存档写清「存了什么」；尽量不采多余个人信息
