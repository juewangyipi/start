========================================
  第一版架构 v0.1（构思层，非代码）
========================================
依据：《构思.txt》已定方向
本文单独维护；不要把本文内容再塞回构思主文档。

版本名：v0.1
定位：只服务「开局可玩」，为以后版本留接口。


----------------------------------------
一、目标（v0.1 要证明什么）
----------------------------------------

要证明：
  · 俯视移动 + 切屏四图能跑通
  · 互动采集（至少河边钓 / 林中砍 一类）
  · 格子背包 + 商店扩容 + 与背包分开的互通仓库
  · 昼夜；死亡回村/镇存档点
  · 本地存档 + 离线补进度（简版）

不证明（不做进 v0.1）：
  · 全技能、完整战斗深度
  · 云存档账号、建城、无缝大地图
  · 大剧情、复杂魔法


----------------------------------------
二、逻辑分层（从上到下）
----------------------------------------

  ┌──────────────────────────────────────────┐
  │  表现层 UI：HUD / 背包 / 仓库 / 商店 / 提示   │
  ├──────────────────────────────────────────┤
  │  场景层：当前地图块 · 实体 · 渲染（像素）      │
  ├──────────────────────────────────────────┤
  │  玩法系统：互动·技能·背包仓库·商店·时间·战斗(可薄)·离线 │
  ├──────────────────────────────────────────┤
  │  世界层：地图注册表 · 切屏 · 存档点 · 碰撞    │
  ├──────────────────────────────────────────┤
  │  数据层：物品 · 技能定义 · 地图数据 · 存档结构  │
  ├──────────────────────────────────────────┤
  │  核心：游戏循环 · 输入 · 配置 · 存读档        │
  └──────────────────────────────────────────┘

原则：
  · 系统尽量不互相硬绑
  · 加新技能 / 新地图优先加「数据」，少改核心


----------------------------------------
三、模块职责（构思名，实现时再落文件）
----------------------------------------

1. 核心 Core
   - 配置：格子大小、版本号 v0.1、键位
   - 循环：更新 → 系统 tick → 渲染
   - 输入：移动、互动、开关背包/仓库

2. 世界 World
   - 地图块（Chunk）：
       id、邻居（上下左右谁接谁）、图块层、碰撞、
       互动物、出生/入口点
   - 地图注册表：village / grassland / riverside / forest
   - 切屏：出边界 → 换块 → 角色放到对侧入口
     （为以后无缝留「块 + 坐标」模型）
   - 存档点：村/镇标记；死亡读「最近激活的存档点」

3. 实体 Entities
   - 玩家：位置（块 id + 块内坐标或世界坐标）、朝向、手持
   - 互动物：树、鱼点、商店、仓库柜、桥/浅滩（类型 + 状态）
   - 怪：v0.1 可 0～1 种（薄），以后再扩

4. 玩法系统 Systems
   - 碰撞：实体 vs 固体格
   - 互动：面向目标 → 砍 / 钓 / 开店 / 开仓
   - 技能：等级、经验、解锁表（v0.1 先 1～2 个生活技能）
   - 背包：格子数组；商店花钱扩容（理论上可一直扩）
   - 仓库：与背包分开；各村镇互通一份数据；可扩容；解锁后通用
   - 商店：卖资源、买扩包/扩仓、基础工具（范围可小）
   - 时间：昼夜相位（氛围为主）
   - 离线：离开前进行中的行动 + 时间戳 → 回来补进度（有上限）
   - 战斗：v0.1 可空壳或极简，接口预留
   - 死亡：回到最近存档点（村落/城镇）

5. 数据 Data（尽量纯数据）
   - items：id、名、堆叠、卖价……
   - skills：id、升级占位、解锁列表
   - maps：四张块布局与出口
   - shop 表、背包/仓库初始格数

6. 存档 Save
   - 存：玩家、背包、仓库、技能、当前地图与坐标、
         昼夜时间、最近存档点、离线时间戳、扩容等级
   - 载：启动读本地；导出/导入预留
   - 云：不进 v0.1；存档结构预留将来可上传

7. UI
   - 常驻：昼夜、金币、简要状态
   - 面板：背包、仓库、商店、技能（可极简）
   - 只通过系统改数据，不直接改地图文件


----------------------------------------
四、开局四图怎么挂
----------------------------------------

示意邻接（方向可调）：

  village ── 草地 grassland ── 河边 riverside ──（过河）── 森林 forest

关键：用「块 id + 邻接表」，不要写死 if (地图名)。

各块职责：
  village    商店、仓库、存档点、出生
  grassland  过渡、少量互动（可选）
  riverside  鱼点；过河点（桥/浅滩）连 forest
  forest     树；稍深一点的探索


----------------------------------------
五、主数据流（一圈）
----------------------------------------

  输入移动 → 碰撞 → 位置变化
  出界 → 切屏换 Chunk
  互动键 → 技能 / 商店 / 仓库
  获得物品 → 背包（满则提示）→ 回村进仓库
  卖出 / 扩容 → 改钱与格数
  每帧或定时 → 昼夜
  关闭或定时 → 写存档 + 时间戳
  再次打开 → 离线补进度 → 进游戏


----------------------------------------
六、扩展点（以后版本，避免推倒）
----------------------------------------

  · 新地图 = 新 Chunk 数据 + 注册邻接
  · 新技能 = 数据 + 一种互动类型
  · 无缝   = 同时加载邻块 + 镜头，Chunk 模型保留
  · 建城   = 新系统，不塞进 v0.1 玩家实体
  · 云存档 = 同一 Save 结构上传
  · 战斗加厚 = 填 Combat 系统，不重写采集

  主线任务系统、全技能、复杂魔法
